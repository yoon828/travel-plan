# PR 작성 예시

Claude는 아래 예시를 참고해서 PR 제목과 본문의 톤, 상세도, 형식을 맞춘다.

---

## 예시 1 — 새 기능 추가 (feat)

**브랜치:** `feat/trip-form`
**제목:** `여행 생성 폼 및 Supabase 연동 구현`

**본문:**
```markdown
## 개요
여행 계획을 시작하려면 기본 정보(제목, 날짜, 멤버)를 한 번에 입력할 수 있는 폼이 필요함
이번 PR에서 여행 생성 폼 컴포넌트를 구현하고 Supabase의 `trips`, `members` 테이블과 연동

---

## 변경 사항

- `TripForm` 컴포넌트 생성 (제목, 날짜 범위, 멤버 닉네임 입력)
- `trips` 테이블 insert 및 `members` 테이블 insert 트랜잭션 처리
- 폼 유효성 검사 추가 (제목 필수, 종료일 > 시작일)
- 생성 완료 후 여행 상세 페이지로 자동 이동

---

## 테스트 방법

1. `npm run dev` 실행 후 http://localhost:3000 접속
2. "새 여행 만들기" 버튼 클릭
3. 제목, 날짜, 멤버 닉네임 입력 후 저장
4. Supabase 대시보드에서 `trips`, `members` 테이블에 데이터 확인

---

## 스크린샷

|--------|-------|
```

## 예시 2 — 버그 수정 (fix)

**브랜치:** `fix/expense-calculation`
**제목:** `지출 정산 시 소수점 반올림 오류 수정`

**본문:**
```markdown
## 개요
지출 금액을 인원수로 나눌 때 소수점이 발생하면 총합이 원래 금액과 달라지는 버그 존재
`Math.floor` 대신 `Math.round`를 적용하고 나머지 금액을 첫 번째 멤버에게 배분하도록 수정

---

## 변경 사항

- `src/lib/budget.ts`의 `splitAmount` 함수 계산 로직 수정
- 나머지 금액(remainder) 처리 로직 추가
- 관련 유틸 함수 단위 테스트 추가

---

## 테스트 방법

1. 지출 항목에 10,000원 / 3명으로 입력
2. 각 멤버 부담액이 3,334 / 3,333 / 3,333 으로 표시되는지 확인
3. 합계가 10,000원인지 확인
```

## 예시 3 — 리팩토링 (refactor)

**브랜치:** `refactor/supabase-query`
**제목:** `Supabase 쿼리 공통 핸들러로 리팩토링`

**본문:**
```markdown
## 개요
각 컴포넌트마다 Supabase 에러 처리 코드가 중복되어 있어 유지보수가 어려웠습니다.
공통 쿼리 핸들러(`supabaseQuery`)를 `src/lib/supabase.ts`에 추가하고 기존 코드를 교체했습니다.

---

## 변경 사항

- `src/lib/supabase.ts`에 `supabaseQuery` 공통 함수 추가
- `TripForm`, `PlaceList`, `ExpenseForm` 컴포넌트의 중복 에러 처리 제거
- 기능 변경 없음, 동작은 기존과 동일

---

## 테스트 방법

1. 여행 생성 / 목적지 추가 / 지출 입력이 기존과 동일하게 동작하는지 확인
2. Supabase 연결 오류 시 에러 메시지가 정상 표시되는지 확인
```

## ❌ 나쁜 PR 예시 (이렇게 하지 말 것)

**제목:** `수정`
**본문:**
```markdown
- 버그 고침
- 코드 정리
```

**왜 나쁜가:**
- 제목만 보고 무슨 작업인지 알 수 없음
- 변경 사항이 구체적이지 않아 리뷰 불가
- 테스트 방법이 없어 리뷰어가 직접 파악해야 함
- 체크리스트 누락
