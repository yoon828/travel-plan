# Git Commit Message Skill

## 역할
변경된 코드(diff 또는 설명)를 분석해서 **이모지 prefix + 한 줄 커밋 메시지**를 생성한다.
메시지는 **무엇을 했는지** 명확하게 전달하는 것이 목표다.

---

## Prefix + 이모지 규칙

형식: `<이모지> <type>: <메시지>`

| type | 이모지 | 언제 사용 |
|------|--------|-----------|
| `feat` | ✨ | 새로운 기능 추가 |
| `feat` | 🌐 | 지도 / 외부 API 연동 작업 |
| `feat` | 💰 | 예산 / 정산 기능 관련 |
| `feat` | 🗺️ | 일정 / 여행 플래닝 기능 관련 |
| `fix` | 🐛 | 버그 수정 |
| `fix` | 🔒 | 보안 수정 |
| `style` | 💄 | UI / 스타일 변경 (기능 변화 없음) |
| `refactor` | ♻️ | 코드 리팩토링 (기능 변화 없음) |
| `refactor` | 🔥 | 코드 / 파일 삭제 |
| `chore` | 🗃️ | DB 스키마 / 마이그레이션 변경 |
| `chore` | 🔧 | 설정 파일 변경 (환경변수, config 등) |
| `chore` | 📦 | 패키지 추가 / 삭제 / 업데이트 |
| `chore` | 🏗️ | 프로젝트 구조 변경 |
| `chore` | 🚀 | 배포 관련 작업 |
| `docs` | 📝 | 문서 / 주석 수정 |
| `test` | 🧪 | 테스트 추가 / 수정 |

---

## 메시지 작성 규칙

### 형식
```
<이모지> <type>: <동사> <무엇을>
```

### 동사 규칙
- **한국어로 작성**
- 과거형 금지 → 현재형 사용
- 명확한 동사 사용

```
✅ 좋은 예
✨ feat: 여행 생성 폼 컴포넌트 추가
🐛 fix: 목적지 순서 변경 시 인덱스 오류 수정
♻️ refactor: Supabase 쿼리 공통 핸들러로 리팩토링
🗃️ chore: expenses 테이블 split_count 컬럼 추가
🔧 chore: Google Maps API 키 환경변수 설정

❌ 나쁜 예
✨ 기능 추가함                    (type 없음, 과거형, 불명확)
🐛 fix: 버그 고침                  (무엇을 고쳤는지 불명확)
✨ feat: 여행 생성 폼 컴포넌트를 추가하고 Supabase trips 테이블과 연동하고 유효성 검사도 추가함 (너무 김)
```

### 길이 제한
- **50자 이내** 권장
- 변경 사항이 많으면 가장 핵심적인 변경 하나만 표현

---

## 변경 사항이 여러 개일 때

변경 내용이 너무 다양하면 하나의 커밋으로 묶지 않고 **분리 커밋을 제안**한다.

```
// 예시: 기능 추가 + 버그 수정이 섞여 있을 때

분리 커밋 추천:
1. ✨ 여행 멤버 등록 기능 추가
2. 🐛 지출 정산 금액 계산 오류 수정
```

---

## 응답 형식

커밋 메시지 요청 시 아래 형식으로 응답한다.

```
## 커밋 메시지

<이모지> <메시지>

---
git commit -m "<이모지> <메시지>"
```

변경 사항이 많아 분리가 필요한 경우:

```
## 커밋 메시지 (분리 권장)

1. <이모지> <메시지>
2. <이모지> <메시지>

분리하지 않을 경우:
<가장 핵심적인 변경을 대표하는 이모지> <메시지>
```

---

## 사용 예시

**요청:**
> "trips 테이블에 destination 컬럼 추가하고 폼에서 입력받도록 수정했어. 커밋 메시지 만들어줘"

**응답:**
```
## 커밋 메시지

🗃️ chore: trips 테이블 destination 컬럼 추가 및 폼 연동

---
git commit -m "🗃️ chore: trips 테이블 destination 컬럼 추가 및 폼 연동"
```

---

## 주의사항
- git diff나 변경 설명 없이 요청하면 어떤 작업을 했는지 먼저 물어볼 것
- 이모지 선택이 애매한 경우 2개 후보를 제시하고 선택하게 할 것
- 절대 메시지를 여러 개 나열해서 고르라고 하지 말 것 → 가장 적합한 것 하나만 제시
